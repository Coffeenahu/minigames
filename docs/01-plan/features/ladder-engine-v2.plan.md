# [PLAN] 정통 사다리 게임 엔진 및 UI 재설계 (개정판)

## 1. 목표
사용자의 요청에 따라 긴장감 있는 1인 순차 진행 방식의 정통 사다리 게임을 구현합니다. 시각적 정합성, 랜덤성, 그리고 결과의 의외성을 극대화하는 것이 목표입니다.

## 2. 사다리 게임 핵심 규칙 (Core Rules)
- **세로선:** 참여자 수(`players.length`)에 맞춰 화면에 균등 배치.
- **가로선(다리):** 
  - 각 게임 시작 시 무작위로 생성.
  - 동일 층에서 연속된 가로선 배치를 금지하여 교차 및 겹침 방지.
- **참여자 및 결과 배치:**
  - **참여자:** 세로선 상단에 입력된 순서대로 배치.
  - **결과:** 세로선 하단에 배치하되, 사용자가 입력한 순서가 아닌 **무작위로 섞어서(Shuffle)** 배치하여 예측 불가능성 부여.

## 3. 구현 프로세스 (Flow)
1. **플레이어 및 결과 입력:** 이름과 결과를 각각 입력받음.
2. **데이터 전처리:** 결과를 무작위로 섞고(Shuffle), 새로운 사다리 난수 데이터 생성.
3. **게임 준비:** 사다리 그림을 화면에 고정 렌더링.
4. **순차적 진행:**
   - 사용자가 참여자를 한 명씩 클릭하여 시작.
   - 해당 참여자의 경로만 **적절한 속도(긴장감 유지)**로 애니메이션 진행.
   - 애니메이션과 사다리 그림이 100% 일치하도록 좌표 기반 드로잉.
5. **결과 확인:** 경로가 끝난 지점의 결과를 팝업으로 표시.

## 4. 기술적 전략 (Technical Strategy)

### 4.1. 랜덤 엔진 (`ladderUtils.js`)
- **결과 섞기:** `shuffleArray` 함수를 추가하여 결과 목록을 게임 시작 직전에 섞음.
- **사다리 생성:** 인접 방지 로직이 적용된 `2차원 배열` 구조 사용.
- **경로 추적:** `(startCol) => points[]` 좌표 배열 생성 시, 꺾이는 지점마다 정확한 좌표를 기록하여 시각적 오차 제거.

### 4.2. 애니메이션 및 UI (`LadderGame.jsx`)
- **1인 순차 진행:** `revealingIndex` 상태를 관리하여 현재 진행 중인 참여자를 추적. 이전 진행이 끝나야 다음 진행이 가능하도록 제어.
- **긴장감 있는 속도:** `polyline`의 `stroke-dashoffset` 애니메이션 속도를 조절하거나, 경로 포인트를 순차적으로 렌더링하여 너무 빠르지 않게 구현.
- **정합성 유지:** 사다리를 그릴 때 사용한 `xOf`, `yOf` 계산식을 경로 애니메이션에서도 동일하게 사용하여 선 위를 정확히 따라가도록 구현.

## 5. 상세 작업 내용
- [ ] **Task 1: 엔진 강화 (Random & Shuffle)**
  - 게임 시작 시 결과 배열을 무작위로 섞는 로직 추가.
  - 인접 가로선 없는 랜덤 사다리 생성 알고리즘 고도화.
- [ ] **Task 2: 사다리 및 결과 배치 시각화**
  - 상단: 참여자, 하단: 랜덤 배치된 결과를 세로선에 맞춰 렌더링.
  - 게임 시작 시 사다리 그림을 먼저 화면에 고정.
- [ ] **Task 3: 긴장감 있는 1인 순차 애니메이션**
  - 한 번에 한 명만 경로를 따라가도록 로직 구현.
  - 경로 이동 속도를 조절하여 사용자가 집중할 수 있도록 구현.
- [ ] **Task 4: 결과 팝업 및 상태 관리**
  - 도달 지점의 결과와 팝업 내용 일치 확인.
  - 전체 결과 보기 기능(진행된 사람들만 또는 전체) 구현.

## 6. 검증 계획
- 사다리 그림의 가로선 위치와 애니메이션 경로가 정확히 겹치는지 확인.
- 결과가 항상 랜덤하게 섞여서 배치되는지 확인.
- 한 명이 이동 중일 때 다른 명의 중복 시작이 방지되는지 확인.
- 모든 참여자가 고유한 결과에 도달하는지 확인.
