# [PLAN] 사다리 게임 V2 개편 계획

## 1. 목표
Stitch 디자인(v2)을 바탕으로 참여자 입력, 결과 입력, 인터랙티브 사다리 게임 및 결과 팝업을 포함하는 멀티 스텝 사다리 게임 흐름을 구현합니다.

## 2. 현황 분석
- **현재 흐름:** `GamePage.jsx` (참여자 입력) -> `LadderGame.jsx` (단일 화면 게임 진행).
- **현재 로직:** `LadderGame`은 `ladderUtils.js`를 사용하여 사다리를 생성하고 경로를 추적합니다. 결과값은 현재 정적입니다(`['당첨 🎉', '꽝 💀', ...]`).
- **새로운 디자인 요구사항:** 
  - 1단계: 참여자 명단 입력 (Home 스타일).
  - 2단계: 결과(벌칙/상품) 입력 (Setup 스타일).
  - 3단계: 인터랙티브 사다리 게임 (Game 스타일).
  - 4단계: 게임 결과 팝업 (Result 스타일).

## 3. 구현 전략
- **내부 단계 관리:** `LadderGame.jsx`에서 `step` 상태를 사용하여 자체적인 흐름을 관리합니다:
  - `STEP_PLAYERS`: 참여자 이름 입력.
  - `STEP_OUTCOMES`: 결과(당첨/벌칙 등) 입력.
  - `STEP_GAME`: 사다리 게임 진행.
- **UI 컴포넌트:** 각 단계별로 새로운 디자인의 스타일(v2 HTML 참조)을 적용한 서브 컴포넌트 또는 렌더링 함수를 생성합니다.
- **데이터 흐름:** 
  - `players`: `GamePage`에서 전달받은 초기값을 사용하되, 1단계에서 수정 가능하도록 구현.
  - `outcomes`: 2단계에서 사용자가 직접 입력.
  - `ladder`: 최종 결정된 `players`와 `outcomes` 수에 맞춰 생성.

## 4. 상세 작업 내용
- [ ] 작업 1: `LadderGame.jsx`에 1단계(참여자 입력) UI 초기화 및 연동.
- [ ] 작업 2: 동적 필드 추가 기능이 포함된 2단계(결과 입력) 구현. (최대 참여자 수 제한 및 미입력 시 '통과' 처리 로직 포함)
- [ ] 작업 3: 사용자 정의 `outcomes`를 사용하도록 게임 엔진 리팩토링. (빈 값을 '통과'로 변환하는 전처리 과정 추가)
- [ ] 작업 4: "Stitch v2" 미학에 맞춰 게임 UI 재설계.
- [ ] 작업 5: 결과 팝업(Result Popup) 오버레이 구현.
- [ ] 작업 6: 최종 테스트 및 마무리.

## 5. 검증 계획

- [ ] 2단계에서 결과 추가/삭제 기능 확인.
- [ ] 결과의 수가 참여자의 수와 일치하는지 확인.
- [ ] 동적 데이터를 사용한 사다리 생성 및 경로 추적 로직 검증.
